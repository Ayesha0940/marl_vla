MARL-VLA Evaluation Instructions (Single Entry Point)
=====================================================

Overview
--------
Use a single command entry point for evaluation:

  python evaluation/eval.py --task <task> --suite <suite> [options]

This guide explains:
1) how to run all evaluation suites from eval.py,
2) suite-specific examples,
3) outputs and expected files,
4) troubleshooting tips.


1) Prerequisites
----------------
- Activate the environment that contains robomimic and project dependencies.
- Run from repository root:

  cd /home/axs0940/marl_vla

- Optional quick check:

  python evaluation/eval.py --list


2) Core CLI Pattern
-------------------
General form:

  python evaluation/eval.py --task <task> --suite <suite> [flags]

Useful common flags:
- --epoch INT
- --n_rollouts INT
- --horizon INT
- --seed INT
- --checkpoint_path PATH
- --checkpoint_name FILE      (square video suite)
- --checkpoint_root PATH
- --run_dir PATH
- --diffusion_model PATH
- --t_start INT [INT ...]
- --render_gpu_id INT
- --best
- --all_runs
- --epochs 100,200,300      (transport sweep)
- --dry_run                 (prints plan without full execution)

5) Dry-Run Mode
---------------
Use dry-run to verify checkpoint resolution, method/noise plans, and transport selection logic:

  python evaluation/eval.py --task can --suite robustness-diffusion --dry_run --epoch 600 \
    --diffusion_model diffusion_models/can_diffusion_model_H1.pt --t_start 10 20

  python evaluation/eval.py --task transport --suite sweep --dry_run --epochs 200,400,600


6) Outputs and Result Files
---------------------------
For robustness and robustness-diffusion suites, eval.py saves:
- JSON summary: results/<task>/robustness_eval_<timestamp>.json
- CSV table:    results/<task>/<task-specific-prefix>_<timestamp>.csv

Current CSV prefixes:
- can:    robustness_eval_can
- lift:   robustness_eval_lift
- square: robustness_eval_square

For filter-sweep suites (can/lift):
- CSV: results/<task>/filter_sweep_<task>_<timestamp>.csv

For robustness-video suites:
- MP4(s) saved under results/<task>/

Transport sweep currently prints ranked summary to console.


7) Migration Status
-------------------
Migration is complete: evaluation execution is centralized in eval.py.
Legacy per-suite evaluation scripts were removed.


8) Recommended Usage Workflow
-----------------------------
Step 1: list suites with --list.
Step 2: test with --dry_run.
Step 3: run full evaluation.
Step 4: check results/<task>/ for CSV/JSON/MP4 outputs.


9) Troubleshooting
------------------
- "robomimic package not found": activate correct conda/env.
- No checkpoint found: verify --epoch and checkpoint directories.
- Vision/offscreen render errors: try --render_gpu_id <id>.
- Need to inspect what would happen without running: add --dry_run.


10) One-Liner Quick Start
-------------------------
List available task/suite combos:

  python evaluation/eval.py --list

Then run:

  python evaluation/eval.py --task <task> --suite <suite> [args]
